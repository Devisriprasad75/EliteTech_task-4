# 🔒 Cloud Security Implementation

## ELiteTech Cloud Computing Internship – Task 4

This project demonstrates the implementation of **Cloud Security** by configuring secure cloud storage with proper access control and data protection techniques.

---

## 🎯 Objective

- Implement secure cloud storage.
- Configure access control using IAM principles.
- Protect files using encryption.
- Demonstrate secure file sharing and storage.

---

## 🛠️ Platform Used

- Google Drive (Cloud Storage Simulation)

---

## 📂 Project Files

- 📄 StudentData.csv
- 📄 SecurityReport.pdf
- 📄 Notes.txt

---

## 🔐 Security Measures Implemented

### ✅ Access Control

- Restricted folder access
- Only authorized users can access the folder
- General Access set to **Restricted**

---

### ✅ IAM (Identity and Access Management)

- Folder ownership assigned to the creator
- Controlled user permissions
- Role-based access (Viewer/Editor)

---

### ✅ Secure Storage

- Files stored securely in cloud storage
- Unauthorized public access disabled
- Sensitive files protected using restricted permissions

---

### ✅ Data Encryption

Google Cloud automatically protects stored data using:

- Encryption at Rest (AES-256)
- Encryption in Transit (TLS/HTTPS)
